# Agent直审 UI 重构设计

**Date:** 2026-04-20

**Status:** Confirmed with user

**Supersedes:** `docs/superpowers/specs/2026-04-18-agent-direct-audit-design.md` 中与前端页面布局相关的部分。本设计不推翻既有直审能力本身，只重构 `Agent直审` 模块的界面组织方式、组件边界和交互规则。

## 目标

将当前 `Agent直审` 页面从“项目上下文 + 会话 + 代码预览 + Inspector 大杂糅三栏”重构为更清晰的双模式工作台：

- `会话管理`
- `项目目录`

新界面要满足以下目标：

- 参考用户提供的 Codex 会话界面和 IDE 代码浏览界面，但保持项目现有的绿色 + 白色视觉体系。
- 明确区分“聊天工作区”和“代码工作区”，避免同屏堆叠过多信息块。
- 让右侧主区域更大，优先服务真实工作流，而不是展示说明卡片。
- 在不破坏现有直审数据流和会话能力的前提下，完成前端结构重构。

## 已确认产品决策

- `Agent直审` 保持为一个独立模块和一个主入口路由，不拆成两个顶层菜单。
- 页面主结构采用 `方案 A`：
  - 顶部模式切换
  - 一个页面内切换 `会话管理` 与 `项目目录`
- `会话管理` 模式采用类似 Codex 的工作区布局：
  - 左侧：按项目组织的会话边栏，可收起/展开
  - 右侧：大聊天窗口
- `会话管理` 模式中：
  - `报告同步` 合并进左侧会话栏底部，作为摘要卡片
  - `Inspector / Findings / Tools / Signals` 暂时从界面中移除
- `项目目录` 模式采用类似 IDE 的工作区布局：
  - 左侧：项目目录树
  - 右侧：代码查看区
- `项目目录` 模式右侧使用 `IDE 标签页` 模式：
  - 点击左侧文件后在顶部打开 tab
  - 支持多个已打开文件之间切换

## 非目标

本次不包含以下内容：

- 不改动 `Agent审计` 模块
- 不新增新的后端数据模型
- 不重新设计会话流协议、文件预览 API 或报告同步逻辑
- 不在本次版本中重新加入 `Inspector / Findings / Tools / Signals`
- 不将 `Agent直审` 拆成两个独立路由页面

## 信息架构

### 页面总结构

`Agent直审` 页面重构为三层结构：

1. 页面壳层
2. 顶部模式切换区
3. 当前模式工作区

建议结构：

- 顶部：标题、简短说明、模式切换器、项目选择器、必要的上下文操作
- 下方：单一主工作台容器
- 工作台内部根据当前模式切换：
  - `会话管理工作区`
  - `项目目录工作区`

页面不再采用“左中右三大功能并列同时可见”的布局，而是让每个模式只承载一种核心任务。

### 模式 1：会话管理

目标是让用户专注于会话与对话。

布局：

- 左栏：
  - 当前项目摘要
  - 会话列表
  - 新建会话入口
  - 左栏收起/展开按钮
  - 底部固定 `报告同步摘要卡片`
- 右侧主区：
  - 会话欢迎态或聊天时间线
  - 用户输入框
  - 流式响应状态

这个模式要尽量接近“左边找会话，右边专注对话”的产品感觉。

### 模式 2：项目目录

目标是让用户专注于浏览源代码。

布局：

- 左栏：
  - 当前项目名
  - 目录树
  - 目录展开/折叠
- 右侧主区：
  - 顶部文件 tab 条
  - 当前激活文件内容
  - 文件空态 / 加载态 / 错误态

这个模式不混入聊天、报告同步摘要和其他分析面板，避免界面再次被功能干扰。

## 状态设计

### 页面根层共享状态

页面壳层保留共享状态：

- `mode`
- `selectedProjectId`
- `projects`
- `sessionIds`
- `projectFiles`

这些状态属于两个模式共享的数据上下文，不应下沉到某一个模式中。

### 会话模式内部状态

建议由会话模式自己维护：

- `selectedSessionId`
- `leftCollapsed`
- `starterPrompt`
- 聊天输入框状态
- 会话创建流式过程状态

说明：

- 左栏收起/展开状态只对会话模式生效。
- 会话模式切走再切回时，尽量保留当前会话和滚动上下文。

### 项目目录模式内部状态

建议由目录模式自己维护：

- `expandedFolders`
- `openTabs`
- `activeTabPath`
- 当前文件请求状态

tab 行为规则：

1. 点击文件，若未打开，则新增一个 tab
2. 点击文件，若已打开，则直接切换到对应 tab
3. 关闭非激活 tab，仅从列表移除
4. 关闭激活 tab，自动切到相邻 tab
5. 所有 tab 关闭后，显示 IDE 空态

### 模式切换保留规则

两个模式都需要遵循以下规则：

- 切换模式时，不清空另一个模式的局部状态
- 同一项目下，用户在两个模式之间切换，应能保留各自工作上下文
- 切换项目时，两个模式的局部状态都要重置，因为数据源已经变化

## 交互规则

### 会话管理模式

- 左栏默认展开，支持一键收起
- 收起后，聊天区自动扩展
- 会话列表按当前项目过滤
- 选中某个会话后，右侧加载对应聊天上下文
- 没有会话时，右侧显示欢迎态和建议输入
- 报告同步能力不再独占右侧区域，而是收纳进左栏底部摘要卡片

摘要卡片建议只保留高价值信息：

- 是否检测到最新报告
- 是否已同步到漏洞管理
- 手动同步按钮
- 自动同步开关

避免在卡片中继续塞入大量说明文本。

### 项目目录模式

- 左侧目录树支持展开/折叠目录
- 点击文件即在右侧打开 tab
- tab 支持切换与关闭
- 当前激活文件在目录树中高亮
- 未打开文件时显示空态，而不是大片说明文案

如果后续需要支持“从聊天消息跳到文件”，这一能力应直接复用 `activeTabPath + openTabs` 机制。

## 视觉设计方向

### 整体风格

采用“绿色白色轻玻璃工作台”风格，而不是仪表盘式信息堆叠。

要求：

- 保持项目现有的浅色背景、绿色主色、白色玻璃卡片和柔和阴影
- 减少大块说明文案
- 减少同屏卡片数量
- 增强主工作区留白
- 强化模式切换和左右工作区的层级感

### 顶部区域

顶部应简洁：

- 标题保留
- 简短说明一句即可
- 主视觉焦点放在模式切换器上

模式切换器建议：

- 激活态：绿色填充或绿色玻璃高亮
- 未激活态：白底浅边框
- 风格与项目主侧边栏保持一致

### 会话管理模式视觉

- 左栏更像轻量工作边栏，而不是信息面板拼盘
- 聊天区作为主画布，视觉面积明显大于左栏
- 底部报告摘要卡片使用更简洁、更收束的卡片样式
- 不出现额外右栏，保证聊天区最大化

### 项目目录模式视觉

- 左侧目录树强调层级和可点击性
- 右侧文件区域更接近 IDE：
  - tab 条清晰
  - 代码内容区边界明确
  - 代码区比当前版本更沉浸
- 视觉仍然保留绿白体系，不做深色 IDE 全量迁移

## 响应式与收缩策略

### 桌面端

- 完整保留双栏工作区
- 会话模式：左栏 + 大聊天区
- 项目目录模式：目录树 + tab + 代码区

### 平板与窄屏

- 左栏默认收起
- 通过按钮展开为抽屉
- 会话模式优先保留聊天区
- 项目目录模式优先保留代码区

本次重构以桌面体验为第一优先，但不能让小屏完全不可用。

## 实施建议

### 推荐实现路径

1. 先拆出页面壳层与顶部模式切换器
2. 再分别抽出 `会话管理` 与 `项目目录` 两个模式组件
3. 迁移现有会话逻辑到 `SessionChatPane`
4. 迁移现有文件树与文件预览逻辑到 `ProjectWorkspaceShell`
5. 增加 tab 状态管理
6. 最后统一调整视觉样式和空态

### URL 与状态同步建议

现有页面已经使用查询参数管理项目、会话、文件。重构后建议：

- 保留 `projectId`
- 会话模式保留 `sessionId`
- 项目目录模式可以保留当前激活文件 `file`
- 可选增加 `mode=session|workspace`

这样刷新后能够恢复到更接近用户离开时的状态。
